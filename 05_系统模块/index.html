什么是系统模块

Node运行环境提供的API，因为这些API都是以模块化的方式进行开发的，所以我们又称Node运行环境提供的API为系统模块。

系统模块 fs 文件操作

f:file文件，s:system系统，文件操作系统。

请求模块：const fs = require('fs');

读取文件内容：fs.readFile('文件路径/文件名称'，'[文件编码]'，callback);

[]表示可选，callback表示文件读取的结果

读取文件语法实例 

	//读取上一层css目录下的base.css
	fs.readFile('../css/base.css,'utf-8',(err,doc) => {
	//如果文件读取发生错误，参数err的值为错误的对象，否则err的值为null;
	//doc参数为文件内容
	if(err == null){
		//在控制台中输出文件内容
		console.log(doc);
	}
	});

系统模块 fs 文件操作

写入文件内容 fs.writeFile('文件路径/文件名称'，'数据'，callback);

写入文件内容实例 

	const content = '
正在使用fs.writeFile写入文件内容。
'
	fs.writeFile('../index.html',content,err => {
		if(err != null){
			console.log(err);
			return;
		}
		console.log('文件写入成功!');
	});

系统模块 path 路径操作（路径拼接API）

为什么要进行路径拼接？

不同操作系统的路径分隔符不统一

Window上是 \ 或 /

Linux上是 /

路径拼接语法 path.join('xxx','yyy',...);

代码实例 

	//导入path模块
	const path = require('path');
	//路径拼接
	let finialPath = path.join('itcast','a','c.css');
	//输出结果 itcast\a\b\c.css
	console.log(finialPath);

相对路径VS绝对路径

大多数情况下使用绝对路径，相对路径有时候相对的是命令行工具的当前目录。

在读取文件或者是指文件路径时都会选择绝对路径

通过__dirname获取当前文件所在的绝对路径
